AKINDO | Guia de Configuração do Firebase
🔑 Configuração necessária

Ativando o login AKINDO

Siga os passos abaixo para conectar o Firebase Authentication ao projeto. São cerca de 5 minutos.

1

Crie o projeto no Firebase Console

Acesse o Firebase Console e crie um novo projeto para o AKINDO.

  • Acesse console.firebase.google.com
  • Clique em "Adicionar projeto"
  • Nome sugerido: akindo-app (pode ser qualquer nome)
  • Google Analytics: opcional, pode desativar
  • Clique em "Criar projeto"
Abrir Firebase Console →
2

Ative o Authentication

No menu lateral esquerdo do Firebase Console:

  • Clique em Authentication
  • Clique em "Vamos começar"
  • Na aba "Método de login", ative:
E-mail/senha → clique → ative a primeira opção → Salvar
Google → clique → ative → preencha o e-mail de suporte → Salvar
3

Registre o app web

Ainda no Firebase Console, vá para as Configurações do projeto (ícone ⚙️ no menu lateral).

  • Role até "Seus aplicativos"
  • Clique no ícone </> (Web)
  • Apelido do app: AKINDO Web
  • Não marque "Firebase Hosting" por enquanto
  • Clique em "Registrar aplicativo"
4

Copie as credenciais e cole no projeto

Após registrar, o Firebase exibirá um bloco de configuração assim:

// Exemplo — seus valores serão diferentes
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain: "meu-projeto.firebaseapp.com",
  projectId: "meu-projeto",
  storageBucket: "meu-projeto.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abc123def456"
};

Abra o arquivo firebase-config.js na raiz do projeto e substitua o conteúdo pelas suas credenciais reais, mantendo o formato abaixo:

/* firebase-config.js — cole suas credenciais aqui */

const firebaseConfig = {
  apiKey: "COLE_SUA_API_KEY_AQUI",
  authDomain: "COLE_SEU_AUTH_DOMAIN",
  projectId: "COLE_SEU_PROJECT_ID",
  storageBucket: "COLE_SEU_STORAGE_BUCKET",
  messagingSenderId: "COLE_SEU_SENDER_ID",
  appId: "COLE_SEU_APP_ID"
};

firebase.initializeApp(firebaseConfig);
const auth = firebase.auth();
5

Autorize o domínio para o login com Google

O Google OAuth só funciona em domínios autorizados. No Firebase Console:

  • Vá em Authentication → Configurações → Domínios autorizados
  • ✓ localhost já está autorizado por padrão
  • Para GitHub Pages, adicione: seu-usuario.github.io
Atenção: O login com Google não funciona ao abrir os arquivos diretamente pelo explorador de arquivos (file://). Use sempre um servidor local (ex: extensão Live Server no VS Code) ou faça o deploy no GitHub Pages.
6

Deploy no GitHub Pages (opcional mas recomendado)

Para apresentar o projeto funcionando 100%, faça o deploy gratuitamente no GitHub Pages:

  • Crie um repositório público no GitHub com o nome akindo
  • Suba todos os arquivos do projeto para a branch main
  • No repositório: Settings → Pages → Branch: main → Save
  • Seu site ficará em: https://seu-usuario.github.io/akindo/html/homepage.html
  • Adicione este domínio no Firebase Console → Authentication → Domínios autorizados

Pronto — login funcionando! 🎉

Com as credenciais configuradas e o domínio autorizado, o cadastro, login por e-mail/senha e login com Google estarão ativos no AKINDO.
